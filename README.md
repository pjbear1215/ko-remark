# rekoit: Remarkable Korean Input Toolkit

reMarkable Paper Pro 및 Paper Pro Move에서 Type Folio와 블루투스 키보드로 한글을 입력할 수 있게 돕는 도구입니다. 전반적으로 CPU와 배터리 소모를 최소화하도록 설계되었습니다.

> **⚠️ 면책 고지 (Disclaimer)**
> - 이 도구는 reMarkable AS의 공식 소프트웨어가 아닙니다.
> - 사용자 소유 기기의 소프트웨어를 수정할 수 있으며, 이로 인해 발생할 수 있는 문제, 데이터 손실, 보증 관련 책임은 전적으로 사용자 본인에게 있습니다.
> - 개발자와 기여자는 기기 변경에 따른 결과에 대해 책임을 지지 않습니다.

## 핵심 기능

### 1. 한글 입력 지원 (Optimized Input)
- **전방위 지원:** 공식 Type Folio와 외장 블루투스 키보드 모두에서 한글 입력을 지원합니다.
- **성능 최적화:** 영문 입력과 대등한 수준의 빠른 타이핑 경험을 제공합니다.
- **한/영 전환:** `Shift + Space` 및 오른쪽 `Alt` 키를 통한 한/영 전환을 지원합니다.

### 2. 블루투스 지원 (Smart Bluetooth)
- **재연결 지원:** Random Address를 사용하는 최신 블루투스 키보드의 자동 재연결을 지원합니다.
- **통합 관리:** 웹 브라우저에서 주변 기기 스캔, 페어링, 연결 로그 확인이 가능합니다.

### 3. 펌웨어 업데이트 대응
- **자동 감지:** 시스템 소프트웨어가 업데이트되어 원본 파일이 바뀌면, 이를 감지하여 최신 버전에 맞는 패치를 자동으로 다시 생성합니다.
- **설정 유지:** 재부팅 후에도 설치 상태가 유지됩니다.

### 4. 웹 기반 관리
- 설치, 블루투스 페어링, 폰트 업로드, 기기 진단 기능을 웹 UI에서 단계별로 제공합니다.

## 1.5 환경별 빠른 가이드

### macOS
- 가장 매끄럽게 동작하는 기본 지원 환경입니다.
- `Homebrew`가 있으면 웹앱의 `사전 준비` 단계에서 필요한 도구를 자동 설치할 수 있습니다.
- 브라우저 자동 열기와 도구 자동 설치까지 한 흐름으로 진행하는 것을 기준으로 맞춰져 있습니다.

### Linux
- 지원되는 패키지 매니저가 있으면 같은 흐름으로 사용할 수 있습니다.
- 비대화형 `sudo`가 가능한 환경이면 웹앱에서 자동 설치를 시도할 수 있습니다.
- 그렇지 않으면 `사전 준비` 단계에서 수동 설치 명령을 복사해 직접 실행한 뒤 다시 확인하면 됩니다.

### WSL
- 기본 흐름은 Linux와 같지만, 도구 설치는 WSL 안에서 진행해야 합니다.
- 브라우저는 가능하면 Windows 쪽으로 자동 열고, 안 되면 `http://localhost:3000` 을 직접 열면 됩니다.
- 패키지 설치 권한이 제한된 WSL 환경에서는 웹앱이 수동 설치 명령을 안내합니다.

설치 도중 재시작이 발생할 수 있으므로 잠금 화면 비밀번호를 꺼두는 편이 안전합니다.

### SSH 비밀번호 확인
다음 위치에서 현재 비밀번호를 확인합니다.
`Settings > General > Help > Copyrights and licenses`

### 연결 준비
- **USB:** 기기와 호스트 PC를 케이블로 연결합니다.
- **무선:** WiFi를 사용하려면 기기 터미널에서 `rm-ssh-over-wlan on` 명령으로 무선 SSH 접속을 미리 허용해야 합니다.

## 3. 호스트 PC에 필요한 것 설치

### Node.js 설치
아직 없다면 먼저 설치합니다.

- 공식 사이트: `https://nodejs.org/`

설치 후 확인:

```bash
node -v
npm -v
```

### 패키지 매니저 확인
자동 설치를 쓰려면 호스트 OS에 맞는 패키지 매니저가 필요합니다.

- macOS: `Homebrew`
- Linux / WSL: `apt`, `dnf`, `pacman`, `zypper`, `apk` 중 하나

예시:

```bash
brew --version
```

Linux / WSL 예시:

```bash
apt-get --version
dnf --version
pacman --version
zypper --version
apk --version
```

### 필수 도구
웹앱은 아래 도구를 확인합니다.

- `ssh`
- `scp`
- `sshpass`
- `zstd`
- `go`
- `zig`

보통 `ssh`와 `scp`는 macOS나 Linux 환경에 이미 포함되어 있습니다. 나머지 도구는 패키지 매니저로 설치할 수 있습니다.

수동 설치 예시:

```bash
# macOS
brew install hudochenkov/sshpass/sshpass
brew install zstd
brew install go
brew install zig

# Debian / Ubuntu / WSL
sudo apt-get update
sudo apt-get install -y openssh-client sshpass zstd golang-go
sudo snap install zig --classic --beta
```

Ubuntu/WSL에서는 `zig` 패키지가 `apt`에 없고, `snap`도 stable 채널에 없을 수 있습니다. 이 경우에는 먼저 아래처럼 시도하세요.

```bash
sudo snap install zig --classic --beta
```

그래도 어렵다면 `ziglang.org`의 공식 바이너리를 직접 내려받아 PATH에 추가하는 방법을 사용해야 합니다.

처음 실행한 뒤 웹앱의 `사전 준비` 단계에서도 설치 상태를 확인할 수 있고, 자동 설치가 어려운 환경에서는 실행해야 할 명령을 그대로 보여줍니다.

### 환경별 사전 준비 단계 동작

#### macOS
- `Homebrew`가 없으면 설치가 필요하다고 안내합니다.
- `Homebrew`가 있으면 웹앱에서 필요한 도구 설치를 바로 진행할 수 있습니다.

#### Linux
- 지원되는 패키지 매니저를 찾으면 그 기준으로 자동 설치 가능 여부를 판단합니다.
- `sudo -n` 이 가능한 환경이면 자동 설치를 시도합니다.
- 관리자 권한 확인이 되지 않으면 수동 설치 명령을 보여줍니다.

#### WSL
- Linux와 같은 기준으로 패키지 매니저와 권한을 확인합니다.
- 도구는 반드시 WSL 내부에 설치되어 있어야 합니다.
- Windows에만 도구가 있고 WSL에 없으면 웹앱에서는 설치되지 않은 것으로 표시됩니다.
- WSL 환경에 `snapd`가 없다면 `zig`는 별도로 수동 설치해야 할 수 있습니다.

## 4. 웹앱 실행

저장소를 별도 웹앱 저장소로 공개했다면, 클론한 저장소 루트에서 아래처럼 실행하세요.

```bash
npm install
npm run dev
```

정상 실행되면 브라우저가 열리거나, 직접 아래 주소로 접속할 수 있습니다.

`http://localhost:3000`

### 페이지가 안 열릴 때

```bash
npm run dev
```

다시 실행한 뒤 브라우저에서 직접 `http://localhost:3000` 으로 접속하세요.

포트가 이미 사용 중이면 다른 프로세스를 끄고 다시 실행해야 합니다.

### 환경별 실행 팁

#### macOS
- `npm run dev` 실행 후 브라우저가 자동으로 열리는 흐름을 기본으로 가정합니다.

#### Linux
- `xdg-open` 이 있으면 브라우저를 자동으로 열려고 시도합니다.
- 자동으로 열리지 않으면 `http://localhost:3000` 을 직접 입력하세요.

#### WSL
- 가능하면 Windows 기본 브라우저로 열려고 시도합니다.
- 자동으로 열리지 않으면 Windows 브라우저에서 `http://localhost:3000` 을 직접 입력하세요.
- 포트 접근이 막히는 환경이라면 WSL 네트워크 설정을 먼저 확인해야 합니다.

## 5. 웹앱에서 실제 설치 순서

### 1단계. 정보 입력 (`Welcome`)
- **IP 주소:** USB(`10.11.99.1`) 또는 WiFi 내부 IP를 입력합니다.
- **SSH 비밀번호:** 기기 설정 메뉴에서 확인한 비밀번호를 입력합니다.
- **SSH 연결 확인:** 접속 테스트 성공 후 `다음` 버튼을 눌러 이동합니다.

### 2단계. 사전 준비 (`Prerequisites`)
웹앱이 호스트 PC의 필수 도구(`ssh`, `sshpass`, `zstd`, `go`, `zig` 등) 설치 상태를 확인합니다.
- 도구가 부족할 경우 자동 설치를 시도하거나 수동 설치 명령을 안내합니다.
- 모든 도구가 준비되면 `다음` 버튼을 누릅니다.

### 3단계. 기기 확인 (`Connection`)
입력된 정보를 바탕으로 기기 연결 상태와 모델 정보를 최종 확인합니다.
- **지원 모델:** Paper Pro 또는 Paper Pro Move인 경우 `감지됨` 표시가 뜹니다.
- 확인이 완료되면 자동으로 다음 단계로 넘어가거나 `다음` 버튼을 누릅니다.

### 4단계. 메뉴 선택 (`Entry`)
기기에서 수행할 작업을 선택합니다.
- **한글 입력 엔진 설치:** 한글 입력 엔진과 필요한 한글 폰트를 자동으로 설치합니다. Type Folio만 사용하는 사용자라면 이 옵션만으로 충분합니다. (필요 시 블루투스 도우미를 추가 선택하여 함께 설치 가능)
- **블루투스 도우미 설치:** 블루투스 도우미만 단독으로 설치합니다. 한글 입력 엔진 없이 영문 입력 환경만 개선할 때 사용합니다.
- **기기 관리:** 이미 설치된 상태에서 블루투스 재페어링, 전원 제어, 키보드 설정만 변경합니다.

### 5단계. 설치 실행 (`Install`)
실제 설치 스크립트가 실행되는 단계입니다.
- 진행 상황이 타임라인 형식으로 표시되며, 완료 후 `블루투스 페어링` 단계로 이어집니다.

## 6. 원상복구 / 제거 관리

rekoit은 시스템 파일을 직접 수정하지 않으므로, 설치된 구성 요소를 제거하고 마운트를 해제하는 것만으로 안전하게 순정 상태로 돌아갈 수 있습니다. 모든 제거 작업은 `4단계. 메뉴 선택(Entry)` 화면에서 **`기기 관리`**를 선택하여 진행합니다.

### 제거 옵션
- **전체 원상복구:** 한글 입력 엔진과 블루투스 도우미를 모두 제거합니다. 단, 설치된 **한글 폰트는 유지**되어 입력 엔진 없이도 한글 문서를 정상적으로 읽을 수 있습니다.
- **부분 제거:** 필요한 경우 `한글 입력 엔진`, `블루투스 도우미`, `한글 폰트`를 각각 개별적으로 제거할 수 있습니다.

### 웹앱으로 제거하는 방법
1. **정보 입력:** IP 주소와 SSH 비밀번호를 입력하고 `SSH 연결 확인`을 통과합니다.
2. **메뉴 선택:** `4단계. 메뉴 선택(Entry)` 화면에서 **`기기 관리`**를 선택합니다.
3. **복구 실행:** 화면 하단의 `복구 및 제거` 섹션에서 원하는 제거 항목의 `시작` 버튼을 누릅니다.
4. **복구 완료:** 모든 구성 요소가 자동으로 제거되고 마운트가 해제됩니다. 작업이 끝나면 해당 기능이 중지되고 기기 설정이 복원됩니다.

### 전체삭제 시 수행되는 작업
- **한글 입력 엔진:** `hangul-daemon` 서비스 중지 및 런타임 파일 제거
- **블루투스 도우미:** 블루투스 관련 서비스 및 페어링 데이터 제거
- **시스템 복구:** `libepaper.so` 등의 `bind mount` 해제 (순정 파일 노출)
- **폰트 제거:** 설치된 한글 폰트 파일 삭제 및 캐시 갱신
- **안전장치 제거:** 펌웨어 업데이트 대응용 포스트 업데이트 훅 제거

### 삭제 후 확인할 것

- 기존 설치 상태가 정리되었는지
- 블루투스 한글 기능이 꺼졌는지
- 필요하면 기기를 한 번 재부팅해도 상태가 유지되는지
- 팩토리리셋을 하지 않았더라도 SSH 비밀번호는 기존 값을 그대로 쓰는지

## 8. 팩토리리셋으로 지우는 방법

웹앱을 쓰지 않고 기기 자체를 초기화하고 싶다면 팩토리리셋을 사용할 수 있습니다.

### 팩토리리셋 시 기대 동작

- 기존 설치 상태 정리
- 블루투스 한글 구성 제거
- 런타임 서비스 / 페어링 데이터 제거
- 한글 폰트도 일반적으로 제거됨

### 팩토리리셋 후 꼭 할 일

1. 기기가 완전히 부팅될 때까지 기다립니다.
2. 새 SSH 비밀번호를 다시 확인합니다.
3. USB를 다시 연결합니다.
4. 필요하면 웹앱의 `기기 관리` 또는 `진단`으로 clean 상태를 다시 확인합니다.

## 9. 팩토리리셋과 펌웨어 업데이트

### 팩토리리셋
팩토리리셋 후에는 설치 흔적이 사라져야 정상입니다. 테스트 기준으로:

- 기존 설치 상태 정리
- 블루투스 한글 구성 제거
- 런타임 서비스 / 페어링 데이터 제거
- SSH 비밀번호 재생성 가능

팩토리리셋 후 다시 접속하려면 SSH 비밀번호를 다시 확인하세요.

### 펌웨어 업데이트
설치 시 SWUpdate post-update 경로와 restore 경로를 준비합니다.

다만 실제 서버에서 어떤 업데이트가 기기에 할당되었는지는 reMarkable 쪽 배포 상태에 따라 달라집니다. 업데이트 후 한글 입력이 정상인지 다시 검증하는 것을 권장합니다.

## 10. 문제 해결

### 웹 페이지가 안 열림

```bash
cd rekoit
npm run dev
```

그래도 자동으로 브라우저가 안 열리면 `http://localhost:3000` 을 직접 입력하세요.

### `sshpass`를 찾을 수 없음

```bash
# macOS
brew install hudochenkov/sshpass/sshpass

# Debian / Ubuntu / WSL
sudo apt-get update && sudo apt-get install -y sshpass

# Fedora
sudo dnf install -y sshpass

# Arch
sudo pacman -Sy --noconfirm sshpass
```

### `zstd`를 찾을 수 없음

```bash
# macOS
brew install zstd

# Debian / Ubuntu / WSL
sudo apt-get update && sudo apt-get install -y zstd

# Fedora
sudo dnf install -y zstd

# Arch
sudo pacman -Sy --noconfirm zstd
```

### `go` 또는 `zig`를 찾을 수 없음

```bash
# macOS
brew install go zig

# Debian / Ubuntu / WSL
sudo apt-get update && sudo apt-get install -y golang-go
sudo snap install zig --classic --beta

# Fedora
sudo dnf install -y golang zig

# Arch
sudo pacman -Sy --noconfirm go zig
```

### SSH 연결 실패
- USB 케이블 연결 확인
- 기기에서 개발자 모드 확인
- 현재 SSH 비밀번호 다시 확인

### 블루투스 패스키가 안 보임
- 키보드를 다시 페어링 모드로 전환
- 같은 키보드가 random 주소로 다시 보일 수 있으므로 재스캔
- 이미 pair/trust가 남아 있으면 전체삭제나 BT 해제를 먼저 실행한 뒤 다시 시도

### 팩토리리셋 후 SSH 접속이 안 됨
- 먼저 기기가 완전히 부팅될 때까지 기다립니다
- SSH 비밀번호를 다시 확인합니다
- USB 연결을 다시 뽑았다가 연결합니다
