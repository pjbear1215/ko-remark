# ko-remark Web Installer

reMarkable Paper Pro / Paper Pro Move에서 Type Folio와 블루투스 키보드로 한글을 입력할 수 있게 해주는 웹 기반 설치 도구입니다.

이 앱은 Mac에서 로컬 웹 서버를 띄운 뒤, USB로 연결된 기기에 SSH로 접속해서 설치를 진행합니다. 설치는 브라우저에서 단계별로 진행되며, Type Folio와 블루투스 키보드용 한글 입력을 설정하거나 전체 원상복구를 실행할 수 있습니다.

## 지원 기기

| 기기 | 코드네임 | 상태 |
|------|----------|------|
| Paper Pro | Ferrari | 지원 |
| Paper Pro Move | Chiappa | 지원 |
| reMarkable 2 | - | 미지원 |

연결 단계에서 기기 모델을 자동으로 감지하고, 지원되지 않는 기기는 설치를 막습니다.

## 할 수 있는 일

### Type Folio / 블루투스 한글 입력
- Type Folio 및 외장 블루투스 키보드로 한글 입력
- 패스키 표시, 페어링, 재연결 지원
- `tmpfs + mmap` 기반 `libepaper` 패치로 런타임 디스크 I/O를 줄임
- 시스템 바이너리(`xochitl`)를 새로 패치하지 않음

### 원상복구 / 관리
- 전체삭제로 설치된 구성 요소 제거
- 한글 폰트도 함께 삭제
- 기존 설치 상태도 함께 정리 가능
- 블루투스 전원 제어, 폰트 업로드, 진단 기능 제공

## 시작 전에 알아둘 점

### 설치 방식
- 현재는 Type Folio / 블루투스 키보드용 한글 입력만 설치합니다.
- 기존 설치 상태가 남아 있다면 전체삭제 후 다시 진행하는 것을 권장합니다.

### 보증 영향
- 현재 설치 경로는 새 `xochitl` 바이너리 패치를 사용하지 않습니다.
- 다만 비공식 수정이므로 사용 전 스스로 판단하는 것을 권장합니다.

### 안전장치
- 전체삭제로 원상복구 가능
- 팩토리리셋 시 설치 흔적 제거를 목표로 동작
- 팩토리리셋 후에는 SSH 비밀번호가 다시 바뀔 수 있음

## 1. 준비물

처음 사용하는 사람 기준으로 필요한 것은 아래와 같습니다.

### Mac 쪽
- macOS
- Node.js 18 이상
- Homebrew
- 인터넷 연결
- USB-C 케이블

### reMarkable 쪽
- reMarkable Paper Pro 또는 Paper Pro Move
- 배터리 충분히 충전
- 개발자 모드 활성화
- 잠금 화면 비밀번호 해제
- 현재 SSH 비밀번호 확인

## 2. 기기에서 먼저 해야 할 일

리마커블에서 아래 순서대로 준비하세요.

### 개발자 모드 켜기
설정에서 개발자 모드를 켭니다. 개발자 모드가 꺼져 있으면 SSH 연결이 되지 않습니다.

### 잠금 비밀번호 끄기
설치 도중 재시작이 발생할 수 있으므로 잠금 화면 비밀번호를 꺼두는 편이 안전합니다.

### SSH 비밀번호 확인
다음 위치에서 현재 비밀번호를 확인합니다.

`Settings > General > Help > Copyrights and licenses`

이 비밀번호는 팩토리리셋 후 바뀔 수 있습니다. 리셋 후에는 항상 다시 확인하세요.

### USB 연결
Mac과 reMarkable을 USB로 연결합니다. 기본 USB 네트워크 주소는 보통 `10.11.99.1` 입니다.

## 3. Mac에 필요한 것 설치

### Node.js 설치
아직 없다면 먼저 설치합니다.

- 공식 사이트: `https://nodejs.org/`

설치 후 확인:

```bash
node -v
npm -v
```

### Homebrew 설치
아직 없다면 설치합니다.

- 공식 사이트: `https://brew.sh/`

설치 후 확인:

```bash
brew --version
```

### 필수 도구
웹앱은 아래 도구를 확인합니다.

- `ssh`
- `scp`
- `sshpass`
- `zstd`
- `go`
- `zig`

보통 `ssh`와 `scp`는 macOS에 기본 포함되어 있습니다. 나머지는 Homebrew로 설치할 수 있습니다.

수동 설치 예시:

```bash
brew install hudochenkov/sshpass/sshpass
brew install zstd
brew install go
brew install zig
```

처음 실행한 뒤 웹앱의 `사전 준비` 단계에서도 설치 상태를 확인할 수 있습니다.

## 4. 웹앱 실행

저장소를 별도 웹앱 저장소로 공개했다면, 클론한 저장소 루트에서 아래처럼 실행하세요.

```bash
npm install
npm run dev
```

정상 실행되면 브라우저가 열리거나, 직접 아래 주소로 접속할 수 있습니다.

`http://localhost:3000`

### 페이지가 안 열릴 때

```bash
npm run dev
```

다시 실행한 뒤 브라우저에서 직접 `http://localhost:3000` 으로 접속하세요.

포트가 이미 사용 중이면 다른 프로세스를 끄고 다시 실행해야 합니다.

## 5. 웹앱에서 실제 설치 순서

처음 사용자라면 아래 순서대로 진행하면 됩니다.

### 1단계. 기기 연결
- IP 주소는 보통 `10.11.99.1`
- SSH 비밀번호는 기기에서 확인한 값 입력
- `USB 연결 확인` 버튼으로 연결 테스트

### 2단계. 사전 준비
웹앱이 필수 도구 설치 여부를 확인합니다.

- Homebrew가 없으면 먼저 설치
- `sshpass`, `zstd`, `go`, `zig`가 없으면 설치

### 3단계. 기기 확인
- SSH 접속이 되는지 확인
- Paper Pro / Paper Pro Move인지 자동 감지

### 4단계. 설치 실행
웹앱이 Type Folio / 블루투스 키보드용 한글 입력 경로를 설치합니다.

설치가 끝나면 `블루투스 페어링` 단계로 이동합니다.

### 5단계. 블루투스 페어링
블루투스 키보드를 사용할 경우:

- 키보드를 페어링 모드로 전환
- 웹앱에서 스캔
- 기기를 선택
- 패스키가 뜨면 키보드에서 입력 후 `Enter`
- 화면에서 입력 완료 확인

### 6단계. 완료 화면 확인
완료 페이지에서 설치 결과를 확인합니다.

- 필요한 검증 항목 통과 여부 확인
- 블루투스 관리
- 폰트 업로드
- 진단 기능 사용 가능

## 6. 원상복구 / 전체삭제

웹앱에서 전체삭제를 실행하면 설치된 상태를 감지해서 해당 항목을 제거합니다.

### 웹앱으로 제거하는 방법

처음 설치할 때와 같은 방식으로 웹앱을 다시 실행합니다.

```bash
git clone https://github.com/bncedgb-glitch/ko-remark.git
cd ko-remark
npm install
npm run dev
```

브라우저에서 `http://localhost:3000` 을 열고 아래 순서대로 진행합니다.

1. 기기를 USB로 연결합니다.
2. 현재 SSH 비밀번호를 입력하고 `USB 연결 확인`을 누릅니다.
3. 첫 화면 오른쪽 위의 `기기 관리`로 들어갑니다.
4. `전체 원상복구`를 선택합니다.
5. `제거 시작`을 누릅니다.

제거가 끝나면 웹앱 로그에서 완료 메시지를 확인하고, 필요하면 다시 기기를 재부팅해 최종 상태를 확인하세요.

### 언제 이 방법을 쓰면 좋은가

- 설치한 항목을 직접 원상복구하고 싶을 때
- 블루투스 페어링 데이터까지 같이 지우고 싶을 때
- 팩토리리셋 전에 먼저 소프트웨어 원복을 하고 싶을 때
- 한글 폰트까지 포함해 완전히 원복하고 싶을 때

### 전체삭제에서 하는 일
- 기존 설치 상태 정리
- `/home/root/bt-keyboard` 제거
- `/opt/bt-keyboard` 제거
- BT 페어링 데이터 제거
- hook / 서비스 / swupdate 관련 파일 제거

### 폰트 처리
현재 `전체 원상복구`는 한글 폰트도 함께 삭제합니다.

### 삭제 후 확인할 것

- 기존 설치 상태가 정리되었는지
- 블루투스 한글 기능이 꺼졌는지
- 필요하면 기기를 한 번 재부팅해도 상태가 유지되는지
- 팩토리리셋을 하지 않았더라도 SSH 비밀번호는 기존 값을 그대로 쓰는지

## 8. 팩토리리셋으로 지우는 방법

웹앱을 쓰지 않고 기기 자체를 초기화하고 싶다면 팩토리리셋을 사용할 수 있습니다.

### 팩토리리셋 시 기대 동작

- 기존 설치 상태 정리
- 블루투스 한글 구성 제거
- hook / 서비스 / 페어링 데이터 제거
- 한글 폰트도 일반적으로 제거됨
- SSH 비밀번호가 새로 생성될 수 있음

### 팩토리리셋 후 꼭 할 일

1. 기기가 완전히 부팅될 때까지 기다립니다.
2. 새 SSH 비밀번호를 다시 확인합니다.
3. USB를 다시 연결합니다.
4. 필요하면 웹앱의 `기기 관리` 또는 `진단`으로 clean 상태를 다시 확인합니다.

## 9. 팩토리리셋과 펌웨어 업데이트

### 팩토리리셋
팩토리리셋 후에는 설치 흔적이 사라져야 정상입니다. 테스트 기준으로:

- 기존 설치 상태 정리
- 블루투스 한글 구성 제거
- hook / 서비스 / 페어링 데이터 제거
- SSH 비밀번호 재생성 가능

팩토리리셋 후 다시 접속하려면 SSH 비밀번호를 다시 확인하세요.

### 펌웨어 업데이트
설치 시 SWUpdate hook와 restore 경로를 준비합니다.

다만 실제 서버에서 어떤 업데이트가 기기에 할당되었는지는 reMarkable 쪽 배포 상태에 따라 달라집니다. 업데이트 후 한글 입력이 정상인지 다시 검증하는 것을 권장합니다.

## 10. 문제 해결

### 웹 페이지가 안 열림

```bash
cd ko-remark
npm run dev
```

그래도 자동으로 브라우저가 안 열리면 `http://localhost:3000` 을 직접 입력하세요.

### `sshpass`를 찾을 수 없음

```bash
brew install hudochenkov/sshpass/sshpass
```

### `zstd`를 찾을 수 없음

```bash
brew install zstd
```

### SSH 연결 실패
- USB 케이블 연결 확인
- 기기에서 개발자 모드 확인
- 현재 SSH 비밀번호 다시 확인
- 팩토리리셋 후에는 비밀번호가 바뀌었을 수 있음

### 블루투스 패스키가 안 보임
- 키보드를 다시 페어링 모드로 전환
- 같은 키보드가 random 주소로 다시 보일 수 있으므로 재스캔
- 이미 pair/trust가 남아 있으면 전체삭제나 BT 해제를 먼저 실행한 뒤 다시 시도

### 팩토리리셋 후 SSH 접속이 안 됨
- 먼저 기기가 완전히 부팅될 때까지 기다립니다
- SSH 비밀번호를 다시 확인합니다
- USB 연결을 다시 뽑았다가 연결합니다
